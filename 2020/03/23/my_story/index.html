    程序思路

    抓取界面上的控件图片，我们先来想一下，如果想要获得界面控件图片那么就需要通过截图来获得当前屏幕内容，对的，这是最简单的方式，也是最笨的方法。我们虽然通过截图得到了当前界面的内容，可这不是我们想要的，我们想要的是截图中的各个小图片，那么这里该怎么获取呢，当前有2种方法，一是通过算法来获取各个小图标的XY值，如果算法不好，会很不稳定，二是通过无障碍来获取当前页面各个控件坐标值，这种方法虽然稳定但是用户在操作上会感觉略微麻烦

    无论是第一种还是第二种首先要解决的就是获取当前屏幕内容，也就是截图，感谢Google，Google在Android SDK 21（Anndroid5.0）上公开了截图API，这让我们不用获取Root(超级管理员权限)也可以得到当前屏幕内容。得到截图后再通过无障碍提供的获取界面控件节点API来得到控件的坐标值，最后再剪切截图得到各个小图片。

Android SDK 21 引入了新的 android.media.projection API，让您可以为应用添加屏幕采集和屏幕共享功能。例如，如果您想截取当前屏幕，便可使用此功能。
“新增的 createVirtualDisplay() 方法允许您的应用将主屏幕（默认显示）的内容采集到一个 Surface 对象中，然后您的应用便可将其发送至整个网络。该 API 只允许采集非安全屏幕内容，不允许采集系统音频。要开始采集屏幕，您的应用必须先使用通过 createScreenCaptureIntent() 方法获得的 Intent 启动屏幕采集对话框，请求用户授予权限。” 摘自谷歌文档
